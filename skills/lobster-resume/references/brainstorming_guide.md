# Brainstorming Guide

Use this guide after a JD is uploaded and before final resume generation. The goal is to help the user remember relevant evidence, not to invent experience.

## Default Offer

After parsing the JD and choosing a template family, ask:

```text
我可以直接生成，也可以先做一轮 5-8 分钟的头脑风暴，帮你从课程、实习、项目、作业、比赛、工具使用里挖出更贴这个 JD 的经历。要不要开始？
```

If the user says no, generate from existing evidence and list what would improve the next version.

If the user says yes, ask focused questions in small batches. Do not ask more than 6 questions at once.

## Question Strategy

Start from JD requirements and search for direct or adjacent evidence:

- Required tools: Have you used this tool, a similar tool, or completed a class/project involving it?
- Responsibilities: Have you done a smaller version of this task in school, internship, research, club work, or personal projects?
- Domain: Have you handled data, documents, users, customers, risk, compliance, content, experiments, or reports in a related context?
- Soft skills: Any example of careful checking, repetitive execution, communication, coordination, or deadline pressure?
- Output artifacts: Did you make slides, Excel files, dashboards, reports, summaries, scripts, notes, or SOPs?
- Scale/metrics: Approximate counts are useful if honest: rows, files, cases, pages, users, frequency, time saved, error reduced.

## Skill Honesty Ladder

When the user has partial knowledge, use honest wording:

- Never used: "了解基础概念", "正在学习", "可快速上手".
- Used in coursework/tutorials: "通过课程/练习掌握基础操作".
- Used in a small project: "在项目中使用 X 完成 Y".
- Used repeatedly: "熟悉 X，能够独立完成 Y".
- Used professionally: "在实习/工作中使用 X 支持 Y".

Do not write "精通", "熟练掌握", "负责搭建", or quantified outcomes unless supported by user evidence.

## Brainstorming Prompts

For AI/data/testing roles:

- 你有没有测试过 ChatGPT、DeepSeek、通义千问或其他 AI 工具？测试过哪些问题、提示词或输出质量？
- 有没有做过数据清洗、分类统计、人工标注、结果对比、Excel 汇总或报告？
- 有没有把一个复杂任务拆成多个测试场景、检查项、步骤表或记录表？
- 有没有英文材料阅读、翻译、整理、指令理解或双语报告经验？
- 有没有用过 Python、SQL、Excel、Stata、SPSS、Tableau、PowerBI 或类似工具？哪怕是课程项目也可以。

For finance/audit/consulting roles:

- 有没有审阅报表、凭证、合同、客户资料、尽调材料或风险信息？
- 有没有做过行业研究、竞品分析、财务模型、估值、路演 PPT 或管理层汇报？
- 有没有在项目里发现异常、追踪原因、整理证据、复核数据？

For marketing/operations/content roles:

- 有没有做过内容策划、活动复盘、用户调研、社媒运营、数据看板、转化分析？
- 有没有从 0 到 1 做过方案、海报、PPT、推文、短视频脚本或商业计划？

For technical/research roles:

- 有没有论文、课程项目、实验、模型、数据集、算法、调参、可视化或代码仓库？
- 有没有把研究方法、变量、样本、结果整理成报告或答辩材料？

## Evidence Conversion

Convert answers into resume-ready evidence:

- Rough answer: "我用 Excel 做过一些统计"
- Better bullet: "使用 Excel 对项目数据进行清洗、分类统计与透视表汇总，形成可复核的数据记录和阶段性报告。"

- Rough answer: "我玩过 ChatGPT"
- Better bullet: "基于 ChatGPT/DeepSeek 等模型进行提示词试验，比较不同问题表述下的输出准确性、逻辑性与实用性。"

- Rough answer: "Python 只会一点"
- Better bullet: "具备 Python 数据处理基础，能够使用 pandas/NumPy 完成表格数据清洗、字段整理与简单统计分析。"

## Update The Profile

After brainstorming, summarize new evidence and ask for confirmation before treating it as saved profile data:

```text
我从这轮头脑风暴里提炼出这些可写进简历的证据，哪些是真实准确的？哪些要删或改弱？
```

Only merge confirmed evidence into the profile.

## Interview Advice

After generating the resume, include concise interview preparation:

- 3-5 likely interview questions based on JD and resume bullets.
- 2-3 strongest stories to prepare with STAR structure.
- Weak spots and honest explanations.
- Skills to review before interview.
- A short self-introduction angle for this JD.

Keep advice practical and tailored to the selected resume version.
